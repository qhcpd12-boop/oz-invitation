기준 문서: [ref/화면설계서.md](ref/화면설계서.md), [ref/개발기획서.md](ref/개발기획서.md), [Figma 시안](https://www.figma.com/design/CUPgvY0YOqis5pZB7hufGP/Full-website-design-Templates-for-Cyprus-University--Community-?node-id=1-1246)

## 1. 기술 스택

| 영역 | 선택 | 비고 |
|------|------|------|
| 프론트엔드 | React 18 + Vite (JS) | SPA |
| UI 라이브러리 | MUI v6 (`@mui/material`, `@mui/icons-material`) | 테마 단일 출처 |
| 라우팅 | `react-router-dom` v6 | `BrowserRouter` + 보호 라우트 |
| 인증 | Firebase Auth (이메일/비밀번호) | 소셜 버튼은 UI만 두고 향후 활성화 |
| DB | Cloud Firestore | onSnapshot 으로 실시간 sync |
| 결제 | Stripe Checkout (PG는 토스페이먼츠 등으로 1:1 치환 가능) | 서버 측에서 금액 결정 |
| 서버리스 | Vercel Functions (`/api/**.js`) | Stripe 세션 생성 + Webhook |
| 배포 | Vercel + Firebase(Firestore Rules/Indexes) | `vercel.json` SPA fallback |

- 데이터 모델
  ```
  invitations/{id}
    ownerId, status (draft|awaiting_payment|paid|published)
    plan, templateId, slug?
    wedding: { groom, bride, parents..., date, time, venue, address, greeting, contact... }
    gallery: [], rsvpEnabled, step (1..4), createdAt, updatedAt, paidAt?, publishedAt?
  ```

- `CreateLayout` — `useInvitationDraft` 훅이 사용자별 단일 draft 보장(`getOrCreateDraft`) + `WizardStepper`

- `StepCheckout` — 요금제 재선택 + 결제 요약 + `POST /api/payments/checkout` (서버 미설정 시 데모 모드)
- `StepComplete` — slug 자동 생성(신랑·신부 한글 정규화) + 발행 + 공개 URL 복사

### Phase 8 · 결제 (서버리스)
- `api/_lib/firebaseAdmin.js` — Admin SDK 싱글턴 (env: `FIREBASE_ADMIN_PROJECT_ID/CLIENT_EMAIL/PRIVATE_KEY`)
- `api/_lib/plans.js` — **서버 측 가격 단일 출처** (클라이언트 값 신뢰 금지)
- `api/payments/checkout.js`
  - 입력 검증 → invitation 존재 확인 → `orders/{orderId}` 'pending' 생성 → Stripe Checkout 세션 생성 → 세션 URL 반환
  - `invitations.status` 를 `awaiting_payment` 로 전이
- `api/payments/webhook.js`
  - `bodyParser:false` + raw body 직접 read → `stripe.webhooks.constructEvent` 서명 검증
  - 이벤트 처리: `checkout.session.completed → paid`, `checkout.session.expired → expired`, `payment_intent.payment_failed → failed`
  - `orders.status`, `invitations.status` 갱신 + 시각 기록

### Phase 9 · 공개 청첩장 + 보안 + 배포
- `/i/:slug` — 셸 미적용, 다크 배경 + 모바일 카드 (헤더 / 인사말 / 예식 안내 / 오시는 길 placeholder)
- `findPublishedBySlug` — `slug + status in [paid, published]` 쿼리
- `firestore.rules`
  - `users/{uid}` 본인만 r/w
  - `invitations/{id}` 소유자 r/w · `paid|published` 만 비로그인 read · status 상승은 클라에서 차단(`draft|awaiting_payment`만 허용)
  - `orders/{id}` 본인 read · write 금지(서버 Admin 전용)
  - `templates/{id}` 누구나 read · write 금지
- `firestore.indexes.json` — 위저드 draft 조회/공개 조회/주문 이력 조회용 복합 인덱스 3종
- `vercel.json` — `/((?!api/).*)` SPA fallback + 함수 maxDuration 15s

## 8. ⚠️ API 구현 베이스 — 명확화 필요

**현재 `api/` 디렉터리는 Cloud Functions(Firebase Functions)가 아니라 Vercel Serverless Functions 기준으로 작성되어 있습니다.** 기획서/플랜에 “Vercel + Serverless Functions” 라고 되어 있어 그 구조로 구현했습니다.

| 항목 | Vercel Functions (현재) | Firebase Cloud Functions (대안) |
|------|------------------------|---------------------------------|
| 파일 위치 | `api/payments/checkout.js` | `functions/src/index.ts` (또는 `.js`) |
| 함수 시그니처 | `export default function handler(req, res)` (Node.js HTTP) | `export const checkout = onRequest(handler)` (`firebase-functions/v2/https`) |
| 환경변수 | Vercel 대시보드 + `process.env.*` | `firebase functions:config:set ...` 또는 `defineSecret` |
| 배포 | `git push` → Vercel 자동 | `firebase deploy --only functions` |
| URL | `https://<vercel>/api/payments/checkout` | `https://<region>-<proj>.cloudfunctions.net/checkout` |
| body parsing | `req.body` 자동(JSON), webhook은 `bodyParser:false` 로 raw 읽기 | `onRequest` 는 기본적으로 raw 가능, `req.rawBody` 제공 |
| 호스팅 통합 | Next/SPA 정적 + `/api` 같은 도메인 자동 프록시 | Hosting rewrites 로 `"function"` 매핑 필요 |

원하시는 베이스가 **Cloud Functions** 라면 다음으로 마이그레이션이 가능합니다:

1. `functions/` 디렉터리 생성 + `firebase init functions` (런타임 Node 20)
2. `api/_lib/firebaseAdmin.js` 는 `import { getFirestore } from 'firebase-admin/firestore'` 그대로 재사용
3. `api/payments/checkout.js` → `functions/src/checkout.js` 로 이동, 핸들러 시그니처를
   ```js
   import { onRequest } from 'firebase-functions/v2/https'
   import { defineSecret } from 'firebase-functions/params'
   const STRIPE_SECRET = defineSecret('STRIPE_SECRET_KEY')
   export const checkout = onRequest({ secrets: [STRIPE_SECRET], cors: true }, async (req, res) => { /* 동일 로직 */ })
   ```
   로 감싸기
4. 웹훅은 `req.rawBody` 사용, Stripe 대시보드 URL 변경
5. 클라이언트 `fetch('/api/payments/checkout')` 대신 함수 URL 사용 또는 `firebase.json` rewrites 로 매핑
   ```json
   { "hosting": { "rewrites": [
     { "source": "/api/payments/checkout", "function": "checkout" },
     { "source": "/api/payments/webhook",  "function": "webhook" }
   ] } }
   ```
6. 프런트는 그대로 Vercel 또는 Firebase Hosting 어디든 배포 가능

**추천**: 현재 스택이 Firebase(Auth/Firestore/Rules) 위주이므로, 단일 콘솔에서 관리·로그·시크릿이 합쳐지는 **Firebase Cloud Functions로 통일**하는 편이 운영상 더 단순합니다. 원하시면 이대로 마이그레이션 작업을 다음 단계로 진행하겠습니다.
